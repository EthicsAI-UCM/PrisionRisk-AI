# Modelo de negocio de PrisonRisk-AI

**Estrategia:** modelo *Open Core ético*  
El núcleo del sistema (GPL-3.0) es libre y auditable. Los ingresos provienen de:
- Integración, despliegue y mantenimiento en centros penitenciarios.  
- Auditorías éticas, formación y soporte técnico.  
- Personalización de módulos específicos bajo contrato institucional.

### Comparativa con modelos no elegidos
| Modelo | Motivo de descarte |
|--------|--------------------|
| **Donaciones / mecenazgo** | Financiación inestable y poco profesionalizable. |
| **Publicidad o monetización de datos** | Incompatible con RGPD y el contexto penitenciario. |
| **Licencia dual (libre + comercial)** | Riesgo de fragmentación de comunidad y lock-in institucional. |

### Riesgos y medidas
- **Lock-in tecnológico:** mitigado mediante código abierto, APIs documentadas y exportabilidad de datos.  
- **Conflictos comunitarios:** uso de gobernanza abierta y auditorías de transparencia.

El modelo garantiza sostenibilidad económica sin comprometer los principios de **ética, libertad de software y soberanía tecnológica**.
